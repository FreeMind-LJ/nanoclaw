# 自选盯盘

当用户希望 NanoClaw 按计划监控一组自选品种，并且只在有意义的触发时提醒，就使用这个技能。

优先使用 NanoClaw 内置调度器和现有 `x-trade` MCP 工具。除非用户明确要求改代码，否则不要新写一套盯盘循环。

## 默认做法

1. 尽量把自选列表规范成 `EXCHANGE.SYMBOL` 形式。
2. 用 `mcp__nanoclaw__schedule_task` 创建周期任务。
3. 在定时任务的 prompt 里，按品种逐个调用 `x-trade` 工具，不要默认全市场扫描。
4. 默认只在触发时提醒；如果没有触发，除非用户要求日报/摘要，否则不发消息。
5. 对触发的品种，先做二次分析，再生成提醒内容。

## 优先使用的工具

- `get_signals`：逐品种首轮筛选，最优先
- `prepare_analysis_context`：当用户希望 AI 直接基于 OHLC、成交量、持仓量和指标做深入解读时优先使用
- `summarize_market_state`：对已触发品种解释趋势、波动、关键位和冲突点
- `build_trade_plan`：只有用户想看入场、止损、止盈建议时才用
- `get_kline_chart`：当用户想结合图形看结构，或需要 AI 图文结合解释时使用
- `scan_market` / `get_alert`：更适合板块或全市场监控，不适合作为小型自选池的默认方案

## 触发后的默认分析深度

如果用户不满足于只看 `forecast` 或方向结论，而是希望 AI 直接解释行情结构，默认补上以下分析：

- OHLC 结构：最近关键 K 线的开高低收、连续阳阴、突破/回落/震荡状态
- 量价关系：放量突破、缩量回踩、放量滞涨、放量下跌
- 持仓量变化：价格涨跌与 `open_oi` / `close_oi` 的同步或背离
- 多周期关系：至少说明日线方向和 `30m` 执行状态是否共振

默认流程：

1. 先用 `get_signals` 做首轮筛选。
2. 对触发品种优先调用 `prepare_analysis_context` 或 `summarize_market_state`。
3. 如果用户明确希望 AI 直接读原始结构，提醒内容里要点名要求分析：
   - 最近 OHLC 结构
   - 成交量变化
   - 持仓量变化
   - 量价持仓是否匹配
   - 当前走势更像趋势强化、回补推动，还是震荡噪音
4. 只有当用户需要具体入场、止损、止盈时，再追加 `build_trade_plan`。

## 默认参数

- 阈值：`abs(forecast) >= 12`
- 频率：
  - 默认按波段节奏做分时段检查，不默认按 `10` 分钟或 `15` 分钟全时段轮询
  - 日盘优先围绕上午开盘后 `30` 分钟、下午开盘后 `30` 分钟、尾盘最后 `30` 分钟安排扫描或提醒
  - 有夜盘的品种，夜盘优先围绕开盘后 `30` 分钟安排扫描或提醒
  - 只有用户明确要求更高频监控时，才改成分钟级轮询
- 输出模式：无触发时静默
- 触发条件：
  - `abs(forecast) >= threshold`
  - 或者用户明确要求每次扫描都汇报

如果用户给了不同规则，以用户为准。

## 盯盘时间默认建议

如果用户没有明确说明交易周期，默认按“波段”盯盘来处理；不要把“全天盯盘”当成推荐方案。

### 波段交易

波段交易默认采用“少盯盘、重触发”的安排：

- 上午开盘后 `30` 分钟重点看一次，确认是否高开低开超预期、板块强弱是否切换。
- 下午开盘后 `30` 分钟再看一次，检查午后资金回流、趋势延续和节奏是否变化；国内期货日盘通常可近似按 `14:00` 左右处理。
- 日盘尾盘最后 `30` 分钟再看一次，用于决定持有、减仓、调止损和次日计划。
- 有夜盘的品种，夜盘开盘后 `30` 分钟再看一次，确认是否出现超预期波动、跳空延续或情绪切换。
- 其余时间默认不持续盯分时，优先依赖条件单、价格提醒、信号触发。

如果用户只想要一个极简时间表，优先给出这些时间点：

- 日盘开盘后 `30` 分钟
- 下午开盘后 `30` 分钟（国内期货日盘通常可近似按 `14:00` 左右处理）
- 日盘尾盘最后 `30` 分钟
- 夜盘开盘后 `30` 分钟（仅适用于有夜盘的品种）

### 中长线

中长线默认不做高频盯盘：

- 开盘看一眼是否出现重大跳空或突发风险。
- 其余时间以日线、周线、仓位和复盘为主，不建议盯分时。
- 只有在触发关键价位、预警条件或重大消息时，才临时加看盘频率。

### 调度建议

给 NanoClaw 建盯盘任务时，默认按交易周期选频率：

- 波段：优先围绕上午开盘后 `30` 分钟、下午开盘后 `30` 分钟、日盘尾盘最后 `30` 分钟、夜盘开盘后 `30` 分钟设置提醒或扫描，不必机械性全天高频轮询。
- 中长线：优先做低频检查和触发式提醒，例如关键价位、止损位、突破位、财报/宏观事件窗口。
- 如果用户明确说“不想一直被打扰”，优先推荐触发式提醒，而不是固定分钟级播报。

## 定时任务 prompt 写法

创建任务时，prompt 要写完整，让后续定时运行的 agent 在脱离当前对话上下文时也能独立执行。

必须包含：

- 精确的自选列表
- 阈值
- 无触发时是否静默
- 是否包含交易计划
- 期待的输出格式

建议使用接近下面这种 prompt：

```text
监控以下自选品种：SHFE.RB、SHFE.HC、DCE.I、CZCE.SA。
对每个品种调用 x-trade 的 get_signals，并传入正确交易所。
只有当 abs(forecast) >= 12 时才继续；如果没有任何品种触发，就不要发送消息。
对每个触发品种，再调用 summarize_market_state；如果需要更细的结构分析，再调用 prepare_analysis_context。
默认盯盘时间：上午开盘后 30 分钟、下午开盘后 30 分钟、日盘尾盘最后 30 分钟；若该品种有夜盘，再加夜盘开盘后 30 分钟。
最终提醒里简洁说明：品种、方向、forecast、最新价、关键位、主要风险，并补充最近 OHLC 结构、量价关系、持仓量变化是否支持当前方向。
如果某个品种具备较强操作性，再补一段简短交易计划摘要。
```

### 模板 1：波段盯盘

```text
监控以下自选品种：SHFE.RB、SHFE.HC、DCE.I、CZCE.SA。
先对每个品种调用 x-trade 的 get_signals，并传入正确交易所。
只有当 abs(forecast) >= 12 时才继续；如果没有任何品种触发，就不要发送消息。

默认盯盘时间按波段节奏执行：
- 上午开盘后 30 分钟
- 下午开盘后 30 分钟
- 日盘尾盘最后 30 分钟
- 若该品种有夜盘，再加夜盘开盘后 30 分钟

对每个触发品种，再调用 summarize_market_state。
如果需要更细的结构分析，再调用 prepare_analysis_context。

最终提醒里简洁说明：
- 品种
- 方向
- forecast
- 最新价
- 关键位
- 主要风险
- 最近 OHLC 结构
- 量价关系
- 持仓量变化是否支持当前方向

如果某个品种具备较强操作性，再补一段简短交易计划摘要；否则不要强行给交易计划。
如果没有任何品种触发，就不要发送消息。
```

### 模板 2：中长线低频检查

```text
低频检查以下自选品种：SHFE.RB、SHFE.HC、DCE.I、CZCE.SA。
先对每个品种调用 x-trade 的 get_signals，并传入正确交易所。

默认只在这些情形继续分析并发送消息：
- abs(forecast) >= 12
- 触发关键价位、止损位、突破位
- 出现明显跳空、趋势破坏或重大风险信号

如果满足条件，再调用 summarize_market_state；
如果用户希望 AI 直接解释结构，再调用 prepare_analysis_context，重点分析 OHLC、量价关系、持仓量变化，以及日线和 30m 是否共振。

最终提醒里简洁说明：
- 品种
- 当前方向
- 日线是否仍然有效
- 关键支撑阻力
- 是否需要调仓、减仓、继续持有或继续等待
- 主要风险

如果没有触发，不要发送消息。
中长线默认不做全天分钟级轮询，优先做触发式提醒或低频检查。
```

## 建任务流程

1. Check whether the user wants:
   - 周期任务
   - 一次性扫描
   - 推送到当前群还是其他已注册群
2. If recurring, create the task with `mcp__nanoclaw__schedule_task`.
3. After scheduling, tell the user:
   - task ID
   - 调度表达式
   - 自选列表
   - 阈值
   - 无触发时是否静默

## 降噪原则

- 自选池场景不要默认全市场扫描，除非用户明确要板块或全市场 sweep。
- 不要反复发送“无信号”消息，除非用户要求摘要。
- 如果用户嫌提醒太多，优先提高阈值或放宽扫描频率，不要先提改代码。
- 不要承诺跨轮次去重，除非系统里已经明确实现。

## 工具缺失时的处理

如果当前 NanoClaw 运行时没有 `x-trade` 工具，就直接说明必须先接入 `x-trade` MCP。不要用网页搜索假装实现市场盯盘。

## 应触发本技能的典型请求

- "帮我盯盘 RB、HC、I，按波段节奏看"
- "做个自选盯盘，上午开盘后半小时、下午开盘后半小时、尾盘和夜盘开盘后看一下"
- "盯一下黑色系，有异动再提醒我，并顺带分析 OHLC、量价和持仓量"
- "Monitor my futures watchlist and alert me on strong signals"
- "Schedule symbol watching for SA, FG, MA and send only trigger alerts"
